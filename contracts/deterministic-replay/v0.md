# Deterministic Replay Contract v0

Status: Accepted
Authority: Research Program Charter v0.2 §8.2
Applies to: X-01 and X-09, and every replay built on their event stream

## Gate

The MVP gate is **Level 1 + Level 2**. Level 3 is recorded but is not required.

- **Level 1 — semantic determinism:** repeated runs have the same event count and order, orders, fills, P&L, and terminal state.
- **Level 2 — hash determinism:** repeated runs have the same SHA-256 of the complete canonical event stream.
- **Level 3 — binary determinism:** output files are byte-for-byte identical. Container metadata and equivalent Parquet metadata may differ, so Level 3 is not an MVP gate.

Matching P&L alone does not satisfy this contract.

## Canonical values

All timestamps are RFC3339 UTC strings ending in `Z`. Price may be integer ticks or a `FixedPointV0`; size, probability, fee, rate, and delay are fixed-point values. Binary floats are invalid. Canonical JSON uses UTF-8, sorted object keys, no insignificant whitespace, and lexically sorted arrays where the schema declares set semantics. `event_id` is `evt_` plus the SHA-256 hex digest of the full validated model dump—including explicit nulls and empty arrays—after removing **only** `event_id`; `payload_sha256` is the prefixed SHA-256 of canonical `payload`.

Runtime timing telemetry is operational data and is excluded from the canonical envelope and hashes.

## Total order

The replay key is the following tagged tuple, in order:

1. `time.receive_at`
2. `(0, time.source_at)` when present, otherwise `(1, "")`
3. `(0, canonical_refs.market_id)` when present, otherwise `(1, "")`
4. `(0, canonical_refs.outcome_id)` when present, otherwise `(1, "")`; this is the canonical outcome/asset identity
5. `payload_sha256`
6. `event_id`

The leading tag makes **NULLS LAST** explicit for every nullable sort field and prevents comparing a null sentinel with a real value. Timestamps compare as validated UTC instants; strings compare by Unicode code point. The Charter fields remain the ordering prefix; `event_id` is the final collision discriminator for otherwise equal events. A reader must issue an **explicit ORDER BY** for the complete key; physical file order and database default order are not contracts.

## Level-2 stream framing

The Level-2 input is not delimiter-based text. It is the exact byte sequence:

1. domain tag `prediction-market:event-stream:v0\x00` encoded as ASCII;
2. one `uint64be` event count;
3. for each event in replay order, the fixed **32-byte** digest decoded from that event's `evt_<64hex>` ID.

The Level-2 stream digest is lowercase `sha256:` plus SHA-256 of that complete frame. Before framing, an implementation must validate every item as `EventEnvelopeV0` and sort the envelopes internally by the complete total-order key; caller iteration order is never hashed silently. The domain tag prevents cross-protocol reuse, the count prevents ambiguous concatenation, and the fixed digest width makes the framing independently implementable.

## Normative semantic validation

Structural JSON/YAML validation alone is insufficient for v0 invariants such as exact probability sums, per-condition rule constraints, lineage, and content hashes. Every YAML contract names `prediction_market.contracts.validate_contract_v0` as its required normative semantic validator. A consumer that cannot run that validator must fail closed rather than accept a structurally valid object.

## Inputs and execution

- Raw objects are immutable and content addressed. Normalized events must be reconstructable from raw objects, raw ordinals, the canonical-ID registry snapshot, the venue-rule snapshot stream, configuration, and the dependency lockfile.
- Every random seed is fixed and stored in the registered experiment card before execution.
- A single writer, or a deterministic partition plan followed by this total-order merge, owns canonical output. Concurrency may not change processing order.
- Each simulated time uses an as-of join against the versioned per-condition venue-rule stream. Every `simulated_order`, `simulated_fill`, and `simulated_pnl` event requires a non-null `rule_snapshot_ref`. Missing or unknown rules fail closed.
- A source gap is a quality event, not an empty interval. An unexplained gap fails the affected reconstruction and blocks dependent backtests.
- Level 1 comparisons include event lineage: every derived event and simulated order must retain its parent event IDs.

## Acceptance evidence

An acceptance record stores both run IDs, the input object hashes, canonical-ID snapshot hash, venue-rule stream hash, configuration hash, lockfile hash, random seed, Level 1 comparison, and Level 2 stream digest. X-09 failure freezes all strategy backtests until the nondeterminism is corrected and the gate reruns.
